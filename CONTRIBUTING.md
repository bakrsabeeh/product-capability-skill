# Contributing

Improvements welcome! Share examples, patterns, anti-patterns, or team experiences.

## How to Contribute

1. Fork this repository
2. Create branch: `git checkout -b feature/your-improvement`
3. Make your changes
4. Commit: `git commit -m "Add: [your improvement]"`
5. Push: `git push origin feature/your-improvement`
6. Open pull request

## What We Welcome

- Real-world capability plan examples
- New constraint categories or patterns
- Anti-patterns from your experience
- Workflow improvements
- Better examples of good vs bad planning
- Integration with other frameworks or methodologies
- Stories of capability planning preventing issues

## Standards

- Include concrete examples from real products
- Use the SKILL.md format for new patterns
- Link to related documentation
- Document trade-offs and tensions
- Be specific and actionable
